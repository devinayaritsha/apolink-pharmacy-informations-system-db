# Apolink - Pharmacy Information System Database

Apolink is a personal database project built using PostgreSQL to simulate pharmacy operational processes, including patient transactions, inventory management, stock monitoring, and reporting.

## Features

- Patient transactions
- Product inventory management
- Supplier management
- Goods receiving transactions
- Stock monitoring
- Stock opname management
- Stock card
- Monthly stock reporting
- Monthly stock opname reporting
- Reporting and analytics

## Database Tables

### Master Data

- pasien
- produk
- supplier

### Transactions

- transaksi
- detail_transaksi
- penerimaan_barang
- detail_penerimaan

### Inventory

- stok_opname
- detail_stok_opname

## Sample Reports

### Transaction Reports

- Patient transaction history
- Detailed patient purchase history
- Product sales report
- Best-selling products report

### Inventory Reports

- Current stock report
- Critical stock report
- Stock card (Kartu Stok)
- Monthly stock report
- Monthly stock opname report

### Supplier Reports

- Goods receiving history
- Supplier purchase report

## Database Concepts Implemented

- Primary Key
- Foreign Key
- One-to-Many Relationships
- Inner Join
- Left Join
- Aggregate Functions (COUNT, SUM)
- GROUP BY
- ORDER BY
- Filtering with WHERE and HAVING
